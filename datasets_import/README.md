# 数据集导入目录

## 使用说明

此目录用于导入已解压的大型数据集，避免通过网页上传大文件导致的长时间等待。

### 使用方法

1. 将您的数据集解压到此目录下的一个子文件夹中
2. 确保数据集符合以下结构：
   - `train/images/` - 训练图像目录
   - `val/images/` - 验证图像目录
   - `test/images/` - 测试图像目录（可选，建议添加以提高模型评估准确性）
   - `train/labels/` - 训练标签目录（可选，如果有标签文件）
   - `val/labels/` - 验证标签目录（可选，如果有标签文件）
   - `test/labels/` - 测试标签目录（可选，如果有标签文件）
   - `classes.txt` - 类别列表文件（可选，每行一个类别名称）

3. 在网页界面中，点击"从本地目录导入"按钮
4. 选择您的数据集目录名称
5. 填写数据集名称和描述，点击导入
6. 可选择自动分割数据集，设置训练集/验证集/测试集的比例

### 数据集分割

系统提供了数据集分割功能，可以在导入时或导入后对数据集进行分割：

1. **导入时分割**：在导入数据集时，勾选“自动分割数据集”选项，设置训练集/验证集/测试集的比例。

2. **导入后分割**：在数据集列表中，点击“分割数据集”按钮，选择分割模式和比例。

分割模式说明：
- **从训练集分割**：从现有训练集中分割出测试集，保留验证集不变。
- **重新分配所有图像**：将所有图像重新分配到训练集、验证集和测试集。

建议的分割比例：
- 训练集：70%
- 验证集：15%
- 测试集：15%

### 测试集的重要性

测试集用于评估模型的最终性能，与验证集的区别在于：

- 验证集用于训练过程中的模型选择和调参
- 测试集仅用于最终评估，不参与训练过程

使用测试集可以更准确地评估模型在真实场景中的性能，减少过拟合风险。

### 注意事项

- 数据集目录名称应避免使用特殊字符和空格
- 图像文件支持JPG、PNG等常见格式
- 标签文件应为YOLO格式的TXT文件
- 对于大型数据集，建议使用此方法而不是通过网页上传
- 建议添加测试集以提高模型评估准确性

使用COCO数据集的详细步骤：

1. 导入COCO数据集
在系统主界面中点击"数据集管理"
点击"导入COCO数据集"按钮
填写数据集名称和描述
上传COCO格式的JSON文件
选择是否自动分割数据集，并设置相应的参数
点击"确定"按钮提交
2. 准备图像文件
在导入COCO数据集后，您需要将图像文件放置到相应的目录中。系统会自动处理带有路径信息的文件名，您只需要关注文件名的最后部分（不包含路径）。

4. 使用导入的数据集进行训练
放置好图像文件后，您可以在系统的训练管理页面中选择该数据集进行训练：

在系统主界面中点击"训练管理"
点击"创建训练任务"按钮
在弹出的模态框中，选择"使用已注册数据集"
从下拉列表中选择您导入的COCO数据集
选择模型类型、设置训练参数
点击"确定"按钮开始训练
注意事项
文件名处理：系统会自动提取文件名的最后部分（不包含路径），您只需要关注文件名本身
图像文件格式：支持的图像格式包括JPG、JPEG、PNG、BMP和WEBP
目录结构：不要修改系统生成的目录结构，否则可能导致训练失败
类别索引：YOLO格式的标签文件中的类别索引是从0开始的，系统会自动进行转换
数据集分割：如果您在导入时选择了自动分割数据集，系统会根据设置的比例将图像分配到训练集、验证集和测试集中
通过这些修改，系统现在能够正确处理带有路径信息的文件名，使您能够更方便地使用COCO格式的数据集进行训练。
